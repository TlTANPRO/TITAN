## 1. Definisi SSOT (SATU Sumber)

**`accounts-full.json` (root)** adalah **single source of truth** untuk data dashboard.

| Properti | Value |
|----------|-------|
| Path | `accounts-full.json` (project root) |
| Git status | **TRACKED** (visible di `git diff`, bisa di-revert) |
| Ukuran V20 | 8.1 MB (9 akun × 4.132 post) |
| Generated by | `scripts/generate-data.mjs` (read scraped → write src/data/) lalu `scripts/deploy.mjs` (copy dist/data/ → root) |
| Disajikan oleh | (1) Vite bundle `assets/accounts-full-{hash}.js`, (2) static fallback `/accounts-full.json`, (3) Worker fetch (jika ada) |
| Frontend baca via | `src/lib/dataStore.js:30` (`import('../data/accounts-full.json')` — Vite auto-resolve) |

**Kenapa root, bukan `scripts/scraped/*.json`?**
- `scraped/*.json` di-`.gitignore` → tidak pernah terlihat di `git diff` → tidak bisa di-revert.
- `accounts-full.json` (root) di-commit → semua perubahan visible + recoverable.
- `scraped/*.json` = **input**, `accounts-full.json` (root) = **deployed output**.

## 2. Pipeline (Alur Tunggal — Tidak Boleh Ada Jalur Lain)

```
scripts/scraped/{slug}.json          ← 9 file, per-akun input, di-gitignore
       │
       │ node scripts/generate-data.mjs
       ↓
src/data/accounts-full.json           ← Vite embed, di-gitignore
       │
       │ pnpm run prebuild (scripts/copy-data-to-public.mjs)
       ↓
public/data/accounts-full.json        ← Vite static + Worker fetch, di-gitignore
       │
       │ pnpm run build
       ↓
dist/data/accounts-full.json          ← Vite built, di-gitignore
       │
       │ pnpm run deploy (scripts/deploy.mjs)
       ↓
accounts-full.json (ROOT)             ← ★ SSOT, TRACKED, DEPLOYED
       │
       │ GitHub Pages auto-rebuild
       ↓
https://tltanpro.github.io/TITAN/     ← LIVE
```

### Satu command untuk deploy
```bash
pnpm run deploy
```
Script `scripts/deploy.mjs` otomatis:
1. Regenerate `src/data/accounts-full.json` dari scraped
2. **Pre-flight**: 9 akun + ≥ 4.000 post + 0 cross-dup — FAIL FAST kalau ada anomali
3. `pnpm run build` (Vite embed + prebuild)
4. Copy `dist/*` ke root (replace `accounts-full.json`, `assets/`)
5. **Cleanup** backup files di `scripts/scraped/` (defense in depth)
6. `git add -A` + commit + push → GitHub Pages

## 3. Aturan Wajib

### 3.1 JANGAN
- ❌ Edit `src/data/accounts-full.json`, `public/data/accounts-full.json`, atau `accounts-full.json` (root) **manual**. Selalu dari `scraped/*.json` → `generate-data.mjs`.
- ❌ Taruh backup file (`*.backup-*.json`) di `scripts/scraped/` — mereka **break `validate-merge.mjs`**. Pakai `git checkout` untuk recovery.
- ❌ Taruh `accounts-full.backup-*.json` di root — recovery via `git checkout`, bukan file backup.
- ❌ Pakai ENSEMBLEDATA — tokens habis. Pipeline saat ini **100% free method**.

### 3.2 LAKUKAN
- ✅ Backup scrape state → `git checkout <commit> -- scripts/scraped/`
- ✅ Backup deployed bundle → `git checkout <commit> -- accounts-full.json assets/`
- ✅ Pakai pattern MAX-merge untuk semua enricher: `if (newVal > existingVal) existing.field = newVal;`
- ✅ Pakai atomic write: `tmp = filepath + '.tmp'; fs.writeFile(tmp, ...); fs.rename(tmp, filepath);`

## 4. Schema Wajib (per akun + per post)

### 4.1 Top-level
```json
{
  "platform": "instagram" | "tiktok",
  "account": {
    "username": "ardiantanah",
    "displayName": "Ardian Tanah",
    "profilePicUrl": "https://...",
    "followerCount": 123456,
    "isVerified": false,
    "localAvatar": "/TITAN/assets/avatars/{slug}.jpg"
  },
  "posts": [ ... ],
  "scrapedAt": "2026-07-19T...",
  "stats": { "totalPosts": 679, "enriched": true }
}
```

### 4.2 Per-post (Instagram)
```json
{
  "id": "3292893687_1234567890",   // format: {userId}_{shortcodeNumeric} — KEEP semua
  "shortcode": "DZK3iOsRlWX",
  "caption": "...",
  "createTime": 1752816000,         // Unix epoch SECONDS
  "timestamp": 1752816000000,       // Unix epoch MS
  "mediaType": "IMAGE" | "VIDEO" | "REEL" | "CAROUSEL_ALBUM",
  "thumbnailUrl": "https://...",
  "postUrl": "https://www.instagram.com/p/{shortcode}/",
  "videoUrl": "https://...",        // empty string jika IMAGE
  "likeCount": 0,                   // 0 = tidak diketahui
  "commentCount": 0,
  "viewCount": 0,                   // 0 untuk IMAGE (no view count)
  "saveCount": 0,                   // IG: 100% tidak achievable tanpa login
  "durationSeconds": 0,
  "hashtags": ["#tag1"],
  "mentions": ["@user1"]
}
```

**Catatan PENTING tentang `id` format `{postId}_{userId}`**:
- Jangan filter berdasarkan `embedded-userId != account.pk` (V19 punya 89 legitimate posts dengan userId mismatch — itu noise dari `/clips/user/` tapi post tetap milik akun yang di-scrape).
- Dedup cukup by `id` atau `shortcode` saja.

### 4.3 Per-post (TikTok)
```json
{
  "id": "7294189237418234567",      // aweme_id (unique)
  "caption": "...",
  "createTime": 1752816000,
  "timestamp": 1752816000000,
  "mediaType": "VIDEO",
  "thumbnailUrl": "https://...",
  "postUrl": "https://www.tiktok.com/@user/video/{id}/",
  "videoUrl": "https://...",
  "likeCount": 0,                   // diggCount di TikWM
  "commentCount": 0,
  "viewCount": 0,                   // playCount di TikWM
  "saveCount": 0,                   // collectCount di TikWM
  "shareCount": 0,                  // TT-specific
  "durationSeconds": 0,
  "hashtags": ["#tag1"],
  "mentions": ["@user1"]
}
```

### 4.4 0 vs null
- Pakai `0` untuk "tidak diketahui" atau "value = 0"
- **JANGAN** `null` atau `undefined` — frontend logic bisa break

| Field | Instagram | TikTok |
|-------|-----------|--------|
| `likeCount` | ✅ Android feed, yt-dlp (REEL/VIDEO only) | ✅ TikWM (100%) |

| `saveCount` | ❌ **0% tanpa login**. Free method tidak ada. | ✅ TikWM (100%) |
| `caption` | ✅ all | ✅ all |
| `thumbnailUrl` | ✅ all | ✅ all |
| `videoUrl` | ✅ all (VIDEO/REEL only) | ✅ all |
| `hashtags` / `mentions` | ✅ all | ✅ all |
| `musicAuthor` / `musicTitle` | n/a | ✅ TikWM |

**Realistic ceiling V20 (per post)**:
- TT like/view/save = 100% (TikWM covers all)
- TT comment = ~95-99% (post deleted di TikTok tidak return data)
- IG like = 70-90% (REEL/VIDEO 100%, IMAGE 0-15%)
- IG view = 60-75% (VIDEO/REEL only, IMAGE 0%)
- IG comment = 50-60%
- IG save = **0%** (dokumentasi: Meta tidak expose `save_count` tanpa GraphQL session)

**Gap yang TIDAK BISA diisi**:
1. IG IMAGE post (~33% IG data): like/view/comment — inherent IG API limit
2. IG saveCount (semua): butuh login session atau ENSEMBLEDATA paid

## 8. Pre-Deploy Checklist (Otomatis di `deploy.mjs` Step 1.5)

Sebelum build + push, `deploy.mjs` validasi:
- ✅ 9 file di `scripts/scraped/*.json` (exclude backup)
- ✅ 9 akun di `src/data/accounts-full.json`
- ✅ Total post ≥ 4.000
- ✅ 0 cross-account duplicate (by shortcode/id)
- ✅ 0 backup file di `scripts/scraped/` (auto-cleanup)

Kalau salah satu gagal → **exit 1, tidak push**. Ini mencegah 0-post deploy seperti bug `validate-merge.mjs` line 120 kemarin.

## 9. Emergency Recovery

### 9.1 Kalau deploy 0 post / data corrupt
```bash
# Lihat history commit
git log --oneline | head -20
# Restore dari commit valid
git checkout <commit> -- accounts-full.json assets/
git commit -m "revert: restore deploy from <commit>"
git push origin main
```

### 9.2 Kalau scraped/ corrupted
```bash
git log --oneline scripts/scraped/ | head -10
git checkout <commit> -- scripts/scraped/
# Re-run pipeline
pnpm run deploy
```

### 9.3 Kalau field coverage turun
- Cek `scripts/coverage-report.mjs` (per-akun × field matrix)
- Cek git log: `git log -p scripts/scraped/ig-ardiantanah.json | head -50`
- Revert: `git checkout HEAD~1 -- scripts/scraped/`

### 9.4 Kalau pre-flight fail terus
- Cek `BACKUP-2026-07-19-pre-cleanup/` (snapshot terakhir pre-cleanup)
- Restore: `cp -r BACKUP-2026-07-19-pre-cleanup/scraped/ scripts/ && pnpm run deploy`
