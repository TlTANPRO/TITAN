## 1. Pipeline Data (Alur Tunggal — tidak boleh ada jalur lain)

```
scripts/scraped/{platform}-{username}.json   ← SATU-SATUNYA sumber
              │                                 (di-gitignore, regenerated by scrapers)
              ↓
   validate-merge.mjs                          (dedupe cross-account, sanity check)
              ↓
   generate-data.mjs                           (aggregate ke src/data/accounts-full.json)
              ↓
   public/data/accounts-full.json               (copy via copy-data-to-public.mjs)
              ↓
   accounts-full.json (root)                   (copy untuk Worker V11 endpoint)
              ↓
   pnpm run build                               (Vite → dist/assets/accounts-full-{hash}.js)
              ↓
   cp dist/assets/* assets/                    (copy bundle ke root assets/)
              ↓
   git add + git commit + git push origin main  (Pages auto-rebuild)
              ↓
   https://tltanpro.github.io/TITAN/            (LIVE)
```

**JANGAN** pernah edit `src/data/accounts-full.json`, `public/data/accounts-full.json`, atau `accounts-full.json` di root **langsung**. Selalu edit dari `scripts/scraped/*.json` lalu re-generate.

## 2. Struktur File Scraped (Schema Wajib)

### 2.1 Top-level (per akun)
```json
{
  "platform": "instagram" | "tiktok",     // WAJIB lowercase, konsisten
  "account": {                             // WAJIB ada untuk validate-merge
    "username": "ardiantanah",             // lowercase, no @
    "displayName": "Ardian Tanah",
    "fullName": "Achmad Ardiansyah",
    "profilePicUrl": "https://...",
    "followerCount": 123456,
    "followingCount": 789,
    "isVerified": false
  },
  "posts": [ ... ],
  "scrapedAt": "2026-07-18T10:00:00.000Z",  // ISO timestamp WAJIB
  "stats": {                                 // metadata scraping run
    "totalPosts": 679,
    "durationMs": 12345,
    "isDummy": false,
    "enriched": true
  }
}
```

### 2.2 Per-post Schema (WAJIB — semua field di bawah harus ada, default 0)

#### Instagram
```json
{
  "id": "3292893687_1234567890",          // WAJIB unique (format: {userId}_{shortcodeNumeric})
  "shortcode": "DZK3iOsRlWX",              // untuk /p/{shortcode} URL
  "caption": "text...",                    // empty string jika no caption
  "createTime": 1752816000,                // Unix epoch seconds (WAJIB)
  "timestamp": 1752816000000,              // Unix epoch ms (WAJIB, = createTime * 1000)
  "mediaType": "IMAGE" | "VIDEO" | "REEL" | "CAROUSEL_ALBUM",  // WAJIB
  "isVideo": true | false,                 // derived dari mediaType
  "thumbnailUrl": "https://...",           // preview image
  "postUrl": "https://www.instagram.com/p/{shortcode}/",  // WAJIB
  "videoUrl": "https://...",               // empty string jika IMAGE/CAROUSEL
  "likeCount": 0,                          // 0 jika tidak diketahui (JANGAN null)
  "commentCount": 0,                       // 0 jika tidak diketahui (JANGAN null)
  "viewCount": 0,                          // 0 untuk IMAGE/CAROUSEL (no view count)
  "playCount": 0,                          // alias viewCount untuk consistency TT
  "saveCount": 0,                          // 0 jika tidak diketahui (IG API limit)
  "durationSeconds": 0,                    // 0 untuk IMAGE
  "hashtags": ["#tag1", "#tag2"],          // array, empty jika none
  "mentions": ["@user1", "@user2"]         // array, empty jika none
}
```

#### TikTok
```json
{
  "id": "7294189237418234567",             // aweme_id (WAJIB unique)
  "caption": "text...",                    // atau "description" (pakai salah satu, consistent)
  "createTime": 1752816000,                // Unix epoch seconds (WAJIB)
  "timestamp": 1752816000000,              // Unix epoch ms (WAJIB)
  "mediaType": "VIDEO",                    // selalu VIDEO untuk TT
  "thumbnailUrl": "https://...",           // cover image
  "postUrl": "https://www.tiktok.com/@user/video/{id}/",  // WAJIB
  "videoUrl": "https://...",               // playUrl (no watermark)
  "likeCount": 0,                          // digg_count di TikWM response
  "commentCount": 0,
  "viewCount": 0,                          // play_count di TikWM response
  "playCount": 0,                          // alias viewCount
  "shareCount": 0,                         // TT-specific
  "collectCount": 0,                       // TT-specific (= saveCount IG)
  "saveCount": 0,                          // = collectCount
  "downloadCount": 0,                      // TT-specific
  "diggCount": 0,                          // = likeCount
  "durationSeconds": 0,
  "duration": 0,                           // ms (TT format)
  "hashtags": ["#tag1"],
  "mentions": ["@user1"],
  "musicAuthor": "Original Sound - user",  // TT-specific
  "musicTitle": "..."
}
```

**PENTING: 0 vs null**
- Gunakan `0` untuk "tidak diketahui" atau "value = 0"
- **JANGAN** gunakan `null` atau `undefined` — buat dashboard logic broken
- Frontend React: `post.likeCount || 0` sudah handle ini, tapi data cleanliness penting untuk validasi

## 4. Aturan Merge (Append-Only / MAX)

Setiap enricher **HARUS** menggunakan pattern:
```js
const existingVal = existingPost.likeCount || 0;
const newVal = response.like_count || 0;
if (newVal > existingVal) existingPost.likeCount = newVal;
```

**JANGAN** pernah overwrite nilai yang lebih tinggi dengan nilai yang lebih rendah.

**PENTING**:
- ID adalah primary key. Post yang sudah ada = update fields. Post baru = append.
- Cross-account dedupe: handled by `validate-merge.mjs` (akun dengan followerCount tertinggi menang)

## 10. Emergency Recovery

### 10.1 Kalau deploy bundle corrupt
- Bundle hash tidak match → `git log --all --oneline | head -20` untuk cari deploy valid
- `git checkout {hash} -- assets/ src/data/ public/data/ accounts-full.json`
- `git commit -m "revert: restore deploy from {hash}"` + push

### 10.2 Kalau field coverage turun
- Cek `scripts/audit-multi-account.mjs` (per-account audit)
- Cek git log: `git log -p scripts/scraped/ig-ardiantanah.json | head -50`
- Revert perubahan: `git checkout HEAD~1 -- scripts/scraped/`

### 10.3 Kalau IG API rate limit permanent
- Switch ke yt-dlp (lebih lambat tapi reliable)
- Atau accept limit (gap IMAGE/save = expected)

## 11. Definisi "Update Data Lengkap" (Kapan Berhenti)

User requirement: "sampai setiap postingan ada itu semua"

**Realistic DoD (Definition of Done)**:
- [ ] Semua posts punya `id`, `shortcode` (IG) atau `aweme_id` (TT), `caption`, `createTime`, `timestamp`, `mediaType`, `postUrl`, `thumbnailUrl`
- [ ] Semua TT posts punya `viewCount` (100% achievable via TikWM)
- [ ] ≥85% IG posts punya `likeCount` & `commentCount` (semua VIDEO/REEL + beberapa IMAGE)
- [ ] ≥55% posts punya `saveCount` (semua TT + IG hanya kalau pakai ENSEMBLEDATA)
- [ ] Tidak ada duplicate post id cross-account
- [ ] 9 akun × total ~4099 posts semua ada
- [ ] Bundle hash berbeda dari deploy sebelumnya (self-validating)
- [ ] Live dashboard render 0 error

**TIDAK achievable dengan FREE methods**:
- ❌ IG IMAGE posts `likeCount/viewCount/commentCount` (33% of IG data)
- ❌ IG `saveCount` untuk semua post

**Untuk achieve 100%**: WAJIB pakai ENSEMBLEDATA paid plan atau IG login session.
